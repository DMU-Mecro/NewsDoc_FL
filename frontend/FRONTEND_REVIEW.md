# Frontend Review Notes

## 1. 담당 기능 및 구현 범위
- 프론트엔드 전체 화면 및 주요 기능 구현 담당
- 구현된 주요 항목:
  - **인증(Auth)**: 로그인 / 회원가입 UI, JWT 저장 및 세션 관리
  - **포트폴리오(Portfolio)**: 자산 등록, 삭제, 자산 비중 차트, 요약 정보
  - **대시보드(Dashboard)**: 마켓 인디케이터, MarketEcho 차트, 전역 상태 관리
  - **리밸런싱(Rebalancing)**: 포트폴리오 비중 비교 및 추천 결과 표시
  - **뉴스 인사이트(Insights)**: Gemini RAG 인사이트, Gemini Pro 리포트 생성
  - **투자 시뮬레이션(Simulation)**: 목표 기반 시나리오별 수익 예측, 차트 표시
  - **설정(Settings)**: 테마, 기간, 지표 토글, 로그아웃

## 2. 구현 상태 요약
### 2.1 완료된 주요 기능
- 로그인/회원가입 화면 및 인증 플로우 구현
- JWT 기반 API 요청 및 401 응답 시 세션 초기화 및 로그인 페이지 리디렉션 처리
- 포트폴리오 자산 등록, 삭제, 차트 시각화, 요약 정보 출력
- 리밸런싱 추천 결과 출력 및 조치 항목 표시
- AI 뉴스 인사이트 조회 및 Gemini Pro 리포트 생성 기능
- 시뮬레이션 결과를 차트로 시각화하는 기능
- 테마, 기간, 지표 토글 등 사용자 설정 기능

### 2.2 보완 필요 항목
- **소셜 로그인은 UI만 구현됨**
  - `frontend/src/components/auth/SocialLogin.jsx`에 버튼만 존재
  - 실제 소셜 OAuth 연동 로직은 아직 미구현
  - 현재는 중요도에 따라 우선순위를 낮춰 둔 상태

- **`useFetch` 훅 미사용**
  - `frontend/src/hooks/useFetch.js`가 존재하지만 컴포넌트에서 호출되지 않음
  - 로그인 기능 개발 과정에서 준비했으나 현재는 적용되지 않음

- **시뮬레이션 입력 변경 시 자동 재계산**
  - `Simulation` 페이지에서 입력값 변경 시 즉시 `runSimulation()`이 호출됨

- **임시 알림 처리**
  - 여러 컴포넌트에서 `alert()`를 사용하여 메시지를 표시함
  - 현재는 임시 구현이며 UI 알림 컴포넌트로 개선 필요

## 3. 개선 작업 계획
- 소셜 로그인 연동 로직 구현 또는 관련 코드 정리
- `useFetch` 훅의 활용 여부 결정 후 코드 정리
- 임시 `alert()` 알림을 UI 컴포넌트로 개선

## 4. 내부 검토 결론
- 프론트엔드 핵심 기능은 대체로 구현되어 있고, 화면 흐름과 API 연동 상태도 양호함
- 다만 **소셜 로그인 미구현, 미사용 훅, 임시 알림 처리**는 후속 보완이 필요함
- 이 문서는 프론트엔드 구현 상태를 정리한 내부 리뷰 노트로 사용함
